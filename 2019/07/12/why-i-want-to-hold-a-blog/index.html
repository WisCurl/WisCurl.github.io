为什么想写一个博客

发表于 2019-07-12 更新于 2019-07-30
本文字数： 1.2k 阅读时长 ≈ 3 分钟

其实没有什么原因，想写就写了。

咳咳，这样说当然太敷衍了。

首先还是想有个能写点东西的地方吧

QQ 空间、微信朋友圈之流当然常用，受众也广，并且算是经过一番预筛选，可一是不是发长篇大论的地方，也就堪几十乃至百余字短言，加上寥寥几张不可排版的配图，让人卡在乘兴与兴尽之间，好生不快；二是生死不由人，嘴巴也不由人，不知何时、何地或许就没了；三是受众虽多，可也攒了一批不甚相识的，若是发些什么都须提心吊胆，细细审核「屏蔽名单」，也忒没意思，再就是「预筛选」这一道门槛，固然拦住了无干之人，可若想把无干变成有干，也得费上好一番工夫。

Telegram 的频道固然很好，缺失的评论功能我也不在意（大可来找我直接交流），它的问题不在于它自身，而在于孤悬海外，一个不受控制的舆论平台。加之功能上太也简略，与其说是发文章不如说是发消息，虽然可以借助 Telegraph，但我就是有点不想用，故弃之，只作为 Qzone, Twitter 之流的替代品。

本来 Evernote 的公开链接分享也是可以的，不过差在零零散散不成系统，也加不了什么更详细的功能，以及毕竟放在别人平台上，除开操于他人之手的问题，蓐羊毛总归是不好的。

所以还是回到承担寄托文章这一功能的自然主体，博客。GitHub 这样的灰色地带就更妙了，既没有被墙之虞（…吧？）,也能拦下一批不大有心思接触新事物及「外面的事物」的人。虽说这一「拦」颇有点防君子不防小人的意味，不过我本来写东西也就是给自己看的，不想大肆传播也不想火，想来当不至于有那么多小人。除非我因为其它原因出名了，嗯。

还有，神夏里面华生的心理医生建议他写博客，我觉得，有些道理。

其次，我又想折腾了

其实倒也不是没考虑过传统的博客平台，国内的不必说，还是那四个字：「操于人手」，加之乌七八糟；WordPress 和 BlogSpot 其实都考虑过，不过一是被墙拦下了，表达欲差了点感觉，就有点不甘心，二是这两个虽无审核，可细究起来还是逃不开「操于人手」（对，最近我对这个很敏感）。那就自己搭吧！正好这时候，我$ \Large\pi$ 推了一篇 GitHub Pages 搭博客的文章，perfect！近期本来就很想捣鼓些 Linux 相关的东西，只是受限于期末一直压着，现在腾出手来，先跟终端好好打打交道，眼熟一下也是好的。敲代码让字符变成漂亮的网页，想想那个 geek 的画面就忍不住蠢蠢欲动。

最后，更方便夹带私货

写在发短消息的平台上，就那么点内容，夹带私货时不免不好意思、犹豫再三甚至干脆取消。博客就不一样了，尽可以放心夹带乃至直接通篇私货，想推书就推书，想写书评就写书评，写书评想剧透就剧透，想安利就安利，哈哈哈哈

最后的最后······不，没最后了。
